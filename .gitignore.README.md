# .gitignore README

## Deskripsi

File `.gitignore` ini dikonfigurasi untuk proyek FitSense Platform yang menggunakan:

- **Node.js** (API Server & Web Dashboard)
- **Python** (ML Service)
- **Docker** (Container orchestration)
- **PostgreSQL, InfluxDB, Redis, EMQX** (Infrastructure)

## Kategori File yang Di-Ignore

### 🔐 Environment & Secrets

- `.env`, `.env.local`, `.env.production`, dll
- File konfigurasi sensitive
- API keys, tokens, credentials

**Exceptions**: `.env.example` tetap di-track sebagai template

### 📦 Dependencies

- `node_modules/` (Node.js)
- `.venv/`, `venv/` (Python virtual environments)
- `__pycache__/` (Python compiled files)

### 🏗️ Build Outputs

- `dist/`, `build/`, `out/`
- `.next/` (Next.js)
- `*.tsbuildinfo` (TypeScript incremental builds)

### 🧪 Testing & Coverage

- `coverage/`, `.nyc_output/`
- `.pytest_cache/`, `.jest-cache/`
- Test reports dan screenshots

### 🗄️ Database & Storage

- `postgres-data/`, `influxdb-data/`, `redis-data/`
- `*.dump`, `*.sql.backup`
- `dump.rdb`, `appendonly.aof` (Redis persistence)

### 📝 Logs

- `*.log`, `logs/`
- Application logs (app.log, error.log, dll)

### 🔒 SSL Certificates

- `*.pem`, `*.key`, `*.crt`
- `nginx/ssl/` directory
- Let's Encrypt certificates

### 📁 Uploads & Media

- `uploads/`, `media/`, `assets/storage/`
- User-generated content

### 💻 IDE & Editor

- `.vscode/` (dengan exceptions untuk shared config)
- `.idea/` (IntelliJ)
- `*.swp`, `*.swo` (Vim)

### 🖥️ Operating System

- `.DS_Store` (macOS)
- `Thumbs.db`, `Desktop.ini` (Windows)
- `.Trash-*` (Linux)

### 🐳 Docker

- `docker-compose.override.yml`
- Local volumes dan data directories

### 📦 Backup & Archive

- `*.backup`, `*.bak`, `*.old`
- `*.tar.gz`, `*.zip`

## File yang TIDAK Di-Ignore (Tracked)

Meskipun ada patterns di `.gitignore`, file-file berikut tetap di-track:

- ✅ `.env.example` - Template environment variables
- ✅ `.vscode/settings.json.example` - VSCode config template
- ✅ `.gitkeep` - Placeholder untuk empty directories
- ✅ `package-lock.json`, `yarn.lock` - Dependency locks (uncomment jika tidak ingin track)

## Cara Menggunakan

### Cek apakah file akan di-ignore

```bash
git check-ignore -v <filename>
```

Contoh:

```bash
git check-ignore -v .env
# Output: .gitignore:8:.env    .env
```

### Lihat semua file yang di-ignore dalam directory

```bash
git status --ignored
```

### Force add file yang di-ignore (jika memang diperlukan)

```bash
git add -f <filename>
```

⚠️ **Warning**: Jangan force add file sensitive seperti `.env` atau credentials!

## Maintenance

### Update .gitignore

Jika ada file baru yang perlu di-ignore:

1. Edit `.gitignore`
2. Tambahkan pattern di kategori yang sesuai
3. Jika file sudah ter-track sebelumnya, jalankan:
   ```bash
   git rm --cached <filename>
   git commit -m "Remove tracked file that should be ignored"
   ```

### Clean ignored files dari working directory

```bash
# Dry run (lihat apa yang akan dihapus)
git clean -ndX

# Hapus semua ignored files
git clean -fdX
```

## Best Practices

1. **Jangan commit file sensitive**: `.env`, SSL certificates, credentials
2. **Gunakan `.env.example`**: Template untuk environment variables tanpa values asli
3. **Track lock files**: `package-lock.json`, `yarn.lock` untuk reproducible builds
4. **Ignore generated files**: Build outputs, coverage reports, logs
5. **Review sebelum commit**: Selalu `git status` untuk memastikan tidak ada file sensitive

## Troubleshooting

### File masih ter-track meskipun sudah di .gitignore

Jika file sudah pernah di-commit sebelum ditambahkan ke `.gitignore`:

```bash
# Remove dari git index (tidak hapus file lokal)
git rm --cached <filename>

# Commit perubahan
git commit -m "Stop tracking <filename>"
```

### Accidentally committed .env

```bash
# Remove dari history (DANGEROUS - koordinasi dengan tim!)
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty --tag-name-filter cat -- --all

# Atau gunakan BFG Repo-Cleaner (recommended)
# https://rtyley.github.io/bfg-repo-cleaner/
```

### Check if .gitignore is working

```bash
# Should not show .env, node_modules, dll
git status --short

# List all ignored files
git status --ignored
```
